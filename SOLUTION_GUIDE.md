# 📱 手機無法訪問解決方案指南

## 🎯 問題確認
- ✅ 電腦可以訪問：http://localhost:8080
- ❌ 手機無法訪問：http://192.168.79.8:8080

## 🚀 推薦解決方案

### 方案 1: 使用 ngrok 隧道（最推薦）

我已經為您安裝了 ngrok，現在只需要幾個步驟：

#### 1. 註冊 ngrok 帳戶
1. 訪問：https://ngrok.com/
2. 註冊免費帳戶
3. 獲取認證令牌

#### 2. 配置 ngrok
```bash
./bin/ngrok config add-authtoken YOUR_TOKEN
```

#### 3. 啟動隧道
```bash
./bin/ngrok http 8080
```

#### 4. 使用 ngrok 網址
ngrok 會顯示類似這樣的網址：
```
https://abc123.ngrok.io
```
在手機瀏覽器中訪問這個網址即可！

### 方案 2: 檢查手機網路設置

#### 常見問題：
1. **WiFi 連接問題**
   - 確保手機和電腦連接同一 WiFi
   - 嘗試重新連接 WiFi
   - 關閉手機 VPN

2. **防火牆問題**
   ```bash
   # 暫時關閉防火牆測試
   sudo pfctl -d
   
   # 測試完成後重新啟用
   sudo pfctl -e
   ```

3. **路由器設置**
   - 檢查路由器是否阻止設備間通信
   - 嘗試重啟路由器

### 方案 3: 使用不同端口

```bash
# 停止當前伺服器 (Ctrl+C)
# 創建使用端口 9000 的伺服器
sed 's/8080/9000/g' server-8080.js > server-9000.js
node server-9000.js

# 在手機上訪問
# http://192.168.79.8:9000
```

### 方案 4: 使用手機熱點

1. 開啟手機熱點
2. 讓電腦連接手機熱點
3. 獲取新的 IP 地址
4. 在手機上訪問新 IP

## 🧪 測試步驟

### 1. 基本測試
在手機瀏覽器中嘗試：
- http://192.168.79.8:8080
- http://192.168.79.8:8080/simple-test.html
- http://192.168.79.8:8080/debug.html

### 2. 網路診斷
```bash
# 檢查網路介面
ifconfig | grep 'inet '

# 檢查端口監聽
netstat -an | grep :8080

# 測試連接
./test-mobile-connection.sh
```

## 🎯 建議操作順序

1. **首先嘗試 ngrok**（最可靠）
2. **檢查手機 WiFi 設置**
3. **嘗試重新連接 WiFi**
4. **檢查防火牆設置**
5. **嘗試不同端口**
6. **使用手機熱點測試**

## 📞 如果問題仍然存在

### 網路環境檢查：
- 是否在企業網路中？
- 路由器是否有特殊設置？
- 是否有網路管理員？

### 替代方案：
- 考慮雲端部署
- 使用 VPS 服務器
- 使用 GitHub Pages 等靜態託管
